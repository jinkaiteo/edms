feat(periodic-review): Implement automatic up-versioning from periodic review

Update periodic review workflow to automatically trigger up-versioning when
changes are required during periodic review.

Changes:
- Update DocumentReview.REVIEW_OUTCOMES from UPDATED/UPVERSIONED to 
  MINOR_UPVERSION/MAJOR_UPVERSION
- Add automatic up-version workflow trigger in complete_periodic_review()
- Update API validation for new outcome choices
- Create migration 0005_update_periodic_review_outcomes

Workflow:
- CONFIRMED: No changes needed (document stays EFFECTIVE)
- MINOR_UPVERSION: Automatically creates v1.0 → v1.1 and starts workflow
- MAJOR_UPVERSION: Automatically creates v1.0 → v2.0 and starts workflow

The new versions are created in DRAFT status and linked to the DocumentReview
record via the new_version field. Original document remains EFFECTIVE until
the new version completes its workflow and becomes EFFECTIVE.
